# IM_project_implementation

The goal of this project was to implement the multi objective production scheduling model from Fechete and Nedelcu (2022) using Python and to verify whether the published results could be independently replicated. The theoretical framework i.e. combining AHP for weight elicitation with a weighted sum MOLP formulation is conceptually sound and is a reasonable approach for balancing conflicting organizational objectives. The Python implementation using scipy.optimize.linprog worked correctly for all four subproblems.

That said, the replication process uncovered a series of issues that made it possible to reproduce the paper’s Table 10 results only when additional constraints are imposed. The most glaring issue was the scaling mismatch in the turnover objective where aggregate monthly revenues were used instead of per unit coefficients. This inflated the sales term by several orders of magnitude relative to the costs and productivity terms, effectively nullifying the AHP weights. On top of this, the October coefficient in Equation (19) appears to be a plain arithmetic error. The published value of 138,455.58 cannot be obtained from the paper’s own formula and data and the correct value (around 162,215) changes the optimizer’s behavior substantially. Finally, the May–June production inversion in Table 10 simply cannot arise from the stated coefficients without additional undocumented constraints.

A fully correct match with Table 10 was only achieved by simultaneously using the paper’s erroneous coefficients and injecting artificial upper bounds on May and October that are not documented in the paper. This was useful in understanding where exactly the results broke down but it also means that Table 10 as published is not reproducible from the paper’s stated methodology.

From an implementation standpoint, this was a very interesting exercise. Working with the constraints and debugging why the solver kept producing October spikes instead of November and tracing the May–June issue back to a coefficient reversal problem all required working with linear programming at a level beyond just plugging numbers in. The AHP computation itself was straightforward once the eigenvector normalization was clear. Overall, the project demonstrated both the utility of multi-objective linear programming as a production planning tool and exercising caution with dimension consistency and constraint correctness when making such models.
